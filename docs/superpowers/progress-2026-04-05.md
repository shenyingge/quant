## 执行期间的发现与修正

1. **QMTCallback vs QMTTrader：** 交易回调方法 `_on_stock_trade_impl` 实际在 `QMTCallback` 类上，而非 `QMTTrader`。计划中的测试模板需要相应调整。

2. **Alembic 幂等性：** 开发 DB 上某些表已由之前的 `metadata.create_all` 创建，migration 需要 inspect() 守卫来避免 `DuplicateTableError`。

3. **Broker ID 空字符串：** 计划模板使用 `str(...) or ""`，审查发现应进一步归一化为 `None`（改为 `str(...) or "" or None`），以便支持 `WHERE broker_trade_id IS NULL` 查询。

4. **filled_time：** 计划模板使用 `datetime.utcnow()`，审查发现应使用 `_extract_trade_timestamp(trade)` 提取的本地变量，确保与 `OrderRecord.filled_time` 一致。

5. **Schema 硬编码：** 迁移文件初版硬编码 `SCHEMA = "trading"`，审查后改为 `get_meta_db_trading_schema()`。

6. **uv 路径：** shell 中 `uv` 命令需使用完整路径 `/c/Users/sai/.local/bin/uv`（已将该路径写入 `~/.bashrc`）。
