# AI Constitution

## Absolute Rules

- 必须先输出 plan，再开始执行。
- 一次最多修改 3 个文件；超过就拆成多步。
- 只允许修改当前任务直接相关的文件。
- 不允许顺手优化、顺手重构、顺手搬迁。
- 涉及行为变化、数据结构、运行流程时，必须先说明风险。
- 改动前必须先读现有实现；高风险路径还要看现有测试或调用链。
- 默认走安全路径，不主动引入真实交易、副作用操作或破坏性变更。
- 完成前必须做与改动直接相关的验证，不能只凭猜测宣称完成。
- 发现范围外问题时只记录，不顺带处理，除非用户明确要求。
- 不允许自动修改architecture.md，AI_CONSTITUTION.md 和 coding-rules.md 这三个核心规范文档。任何对这些文档的修改都必须经过人工审核和确认，确保规范的稳定性和一致性。对于需要修改这些文档的情况，应该先在相关的 issue 或 pull request 中提出修改建议，并进行讨论，直到达成共识后再进行修改。这样可以避免不必要的变动，保持项目规范的权威性和可靠性。
